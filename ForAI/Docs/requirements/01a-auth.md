# 认证 & 用户资料模块需求文档

---

## 1. 模块概述

本模块覆盖用户身份认证和用户个人资料管理，包含 4 个页面：

| 路由 | 页面 | 需要登录 |
| --- | --- | --- |
| `/login` | 登录 | 否（已登录跳转首页） |
| `/register` | 注册 | 否（已登录跳转首页） |
| `/me` | 个人中心 | 是 |
| `/me/profile` | 编辑个人资料 | 是 |

> 忘记密码由用户联系管理员处理，初期不做自助找回。

---

## 2. 注册流程

### 2.1 路径

`/register`

### 2.2 用户操作

1. 用户填写用户名、邮箱、密码
2. 提交注册表单
3. 成功后跳转登录页或直接登录

### 2.3 校验规则

- 用户名：必填，2-40 字符，大小写不敏感唯一
- 邮箱：必填，合法邮箱格式，大小写不敏感唯一
- 密码：必填，最小 8 位

### 2.4 错误处理

- 用户名或邮箱已存在 → 提示"用户名或邮箱已被注册"（不区分是哪个冲突）
- 格式不合法 → 对应字段红色提示
- 网络错误 → 提示重试

### 2.5 页面状态

| 状态 | 展示 |
| --- | --- |
| 正常 | 注册表单 |
| 已登录 | 跳转首页 |
| 提交中 | 按钮 loading，禁用输入 |
| 成功 | 跳转登录页或直接登录 |
| 错误 | 对应字段或全局错误提示 |

---

## 3. 登录流程

### 3.1 路径

`/login`

### 3.2 用户操作

1. 用户填写邮箱、密码
2. 提交登录
3. 后端签发 JWT，通过 httpOnly Cookie 返回
4. 前端跳转首页或来源页

### 3.3 校验规则

- 邮箱：必填
- 密码：必填

### 3.4 错误处理

- 登录失败 → 统一提示"邮箱或密码错误"（不区分邮箱不存在和密码错误）
- 账号被禁用 → 提示"账号已被禁用"

### 3.5 页面状态

| 状态 | 展示 |
| --- | --- |
| 正常 | 登录表单 + 注册入口 |
| 已登录 | 跳转首页 |
| 提交中 | 按钮 loading |
| 401 | 统一错误提示 |
| 403 | 账号禁用提示 |

---

## 4. 退出登录

### 4.1 操作

用户点击退出 → 后端清除 Cookie → 前端跳转首页

### 4.2 约束

- 退出接口必须清除 httpOnly Cookie
- 退出后前端登录态立即失效

---

## 5. 个人中心 `/me`

### 5.1 展示内容

- 用户头像
- 用户名
- 学校/公司信息（如已填写）
- Bio 简介（如已填写）
- 功能入口：编辑资料、我的投稿、我的草稿、我的评论、我的通知、投稿
- 未读通知数量

### 5.2 页面状态

| 状态 | 展示 |
| --- | --- |
| 正常 | 用户信息 + 入口列表 |
| 未登录 | 跳转登录页 |
| 加载中 | 骨架屏 |
| 加载失败 | 错误提示 + 重试 |

---

## 6. 编辑个人资料 `/me/profile`

### 6.1 功能

- 上传/更换头像
- 编辑 Bio（文本，限 200 字）
- 编辑学校/公司（文本，各自限 100 字）

### 6.2 头像规则

- **自定义上传**：用户可上传图片作为头像（仅限 jpeg/png/webp，≤2MB）
- **默认头像**：未上传时自动生成（类似 GitHub identicon，基于用户名 hash 生成几何图案）
- **裁剪**：上传后支持简单裁剪为正方形（TODO：是否 MVP 做裁剪确认）

### 6.3 头像展示位置

头像在以下位置展示：

1. 文章卡片作者区
2. 文章详情页作者信息区
3. 评论区每条评论旁
4. 个人中心 `/me`
5. 通知列表触发人头像
6. 管理后台用户列表

### 6.4 Bio 规则

- 仅 `/me` 个人中心展示
- 最长 200 字符
- 支持纯文本，不支持 Markdown 或 HTML

### 6.5 学校/公司规则

- 自填展示，不做真实性校验
- "学校"和"公司"为两个独立字段
- 填写后展示在 `/me` 个人中心
- 非必填

### 6.6 页面状态

| 状态 | 展示 |
| --- | --- |
| 正常 | 头像区 + 各编辑字段 + 保存按钮 |
| 未登录 | 跳转登录页 |
| 头像上传中 | 上传进度或 loading |
| 头像上传失败 | 文件校验错误提示 |
| 保存中 | 按钮 loading |
| 保存成功 | 提示"已保存" |
| 保存失败 | 对应字段错误提示 |

---

## 7. 权限约束

- 所有认证接口（注册、登录）仅 guest 可访问
- 已登录用户访问登录/注册页时跳转首页
- `/me` 和 `/me/profile` 必须登录
- 用户只能编辑自己的资料
- 前端不读取 Cookie，JWT 仅存 httpOnly Cookie
- 禁止前端将 JWT 存到 localStorage/sessionStorage
- 忘记密码由用户联系管理员处理

---

## 8. 验收标准

### 注册

- [ ] 用户可用有效信息注册成功
- [ ] 用户名或邮箱重复时提示错误
- [ ] 密码不足 8 位时提示错误
- [ ] 已登录用户访问注册页跳转首页

### 登录

- [ ] 用户可用正确凭据登录
- [ ] 错误凭据时统一提示"邮箱或密码错误"
- [ ] 禁用账号登录时提示"账号已被禁用"
- [ ] 登录后 Cookie 为 httpOnly
- [ ] 已登录用户访问登录页跳转首页

### 退出

- [ ] 退出后 Cookie 被清除
- [ ] 退出后访问需登录页面跳转登录页

### 个人中心

- [ ] 未登录用户访问 `/me` 跳转登录页
- [ ] 已登录用户可看到个人信息和功能入口
- [ ] 加载失败时展示重试

### 编辑资料

- [ ] 用户可上传自定义头像
- [ ] 未上传头像时展示自动生成的默认头像
- [ ] 用户可编辑 Bio（最长 200 字）
- [ ] 用户可编辑学校和公司信息
- [ ] 头像在六处展示位置均正确显示
- [ ] 未登录用户访问 `/me/profile` 跳转登录页
