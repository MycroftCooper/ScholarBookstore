# 管理后台模块需求文档

文章审核、内容管理、版块/领域管理、用户管理、举报管理、数据看板、Tag 管理。

---

## 1. 模块概述

管理后台采用**子路由**结构，共 10 个子路由：

| 路由 | 页面 | 角色 |
| --- | --- | --- |
| `/admin` | 后台首页（重定向到 `/admin/reviews`） | reviewer+ |
| `/admin/reviews` | 文章审核 | reviewer+ |
| `/admin/articles` | 文章内容管理 | reviewer+ |
| `/admin/comments` | 评论管理 | reviewer+ |
| `/admin/modules` | 版块管理 | admin |
| `/admin/domains` | 领域管理 | admin |
| `/admin/users` | 用户管理 | admin |
| `/admin/tags` | Tag 管理 | admin |
| `/admin/reports` | 举报管理 | reviewer+ |
| `/admin/dashboard` | 数据看板 | reviewer+ |

**通用要求**：
- 每个子页面有加载状态、空状态、错误状态
- 权限不足时展示 403
- 未登录跳转登录页
- 移动端可访问，但优先保证桌面体验

## 2. `/admin/reviews` — 文章审核

### 2.1 功能

审核员审核用户投稿。

### 2.2 待审核列表

- 展示所有 `pending_review` 状态文章（分页）
- 每项展示：标题、作者（头像+用户名）、所属版块、提交时间
- 点击进入审核详情

### 2.3 审核操作

审核详情展示：
- 文章完整信息（标题、摘要、正文 Markdown 渲染、Tag）
- 审核操作区

**审核通过**：
1. 审核员点击"通过"
2. 可选填写审核说明
3. 文章状态 → `published`，记录审核人和时间

**审核拒绝**：
1. 审核员点击"拒绝"
2. **必填**拒绝原因（`review_note`）
3. 文章状态 → `rejected`

### 2.4 修订审核

对于已发布文章的修订版（`pending_review` + 关联原文）：
- 审核详情可对比原文与修订内容
- 通过 → 修订版替换原文，版本数 +1
- 拒绝 → 修订版变 `rejected`，原文不变

### 2.5 页面状态

| 状态 | 展示 |
| --- | --- |
| 正常 | 待审核列表（分页） |
| 空状态 | "暂无待审核投稿" |
| 加载失败 | 错误提示 + 重试 |
| 无权访问 | 403 |

## 3. `/admin/articles` — 文章内容管理

### 3.1 功能

管理所有文章（不限状态），支持内容下架与恢复。

### 3.2 文章列表

- 展示所有文章（分页）
- 筛选：按状态（all / published / archived / rejected / pending_review / draft）
- 搜索：按标题或作者搜索
- 每项展示：标题、作者、状态、所属版块、发布时间、浏览量
- 操作：查看详情、下架（`published` → `archived`）、恢复（`archived` → `published`）

### 3.3 下架与恢复

| 操作 | 状态变化 | 说明 |
| --- | --- | --- |
| 下架 | `published` → `archived` | 文章不再公开可见 |
| 恢复 | `archived` → `published` | 文章重新公开可见 |

### 3.4 页面状态

| 状态 | 展示 |
| --- | --- |
| 正常 | 文章列表（分页+筛选） |
| 空状态 | "暂无文章" |
| 加载失败 | 错误提示 + 重试 |
| 无权访问 | 403 |

## 4. `/admin/comments` — 评论管理

### 4.1 功能

管理所有评论和回复。

### 4.2 评论列表

- 展示所有可见和已隐藏评论（不包含已删除）
- 筛选：按可见状态（visible / hidden）
- 每项展示：评论内容摘要、评论者、所属文章、发布时间、可见状态
- 操作：隐藏、恢复、删除

### 4.3 操作

| 操作 | 效果 |
| --- | --- |
| 隐藏 | 评论 `visibility` → `hidden`，普通用户不可见 |
| 恢复 | 评论 `visibility` → `visible` |
| 删除 | 软删除评论（设置 `deleted_at`） |

### 4.4 页面状态

| 状态 | 展示 |
| --- | --- |
| 正常 | 评论列表（分页+筛选） |
| 空状态 | "暂无评论" |
| 加载失败 | 错误提示 + 重试 |
| 无权访问 | 403 |

## 5. `/admin/modules` — 版块管理

### 5.1 功能

admin 管理版块（仅 admin 可访问）。

### 5.2 版块列表

- 展示所有版块（按所属领域 + `sort_order` 排序）
- 每项展示：名称、slug、所属领域、排序、启用状态
- 操作：创建、编辑、启用/禁用

### 5.3 创建版块

- 必填：名称、slug、所属领域
- 选填：描述、排序号
- slug 必须唯一
- 创建后默认启用

### 5.4 编辑版块

- 可修改：名称、描述、排序号、启用状态
- 不可修改：slug、所属领域

### 5.5 页面状态

| 状态 | 展示 |
| --- | --- |
| 正常 | 版块列表 |
| 空状态 | "暂无边块" |
| 创建表单 | 弹窗或独立表单页 |
| slug 冲突 | 提示"标识已被使用" |
| 无权访问 | 403（非 admin） |

## 6. `/admin/domains` — 领域管理

### 6.1 功能

admin 管理领域（仅 admin 可访问）。

### 6.2 领域列表

- 展示所有领域（按 `sort_order` 排序）
- 每项展示：名称、版块数量、启用状态
- 操作：创建、编辑、启用/禁用

### 6.3 创建领域

- 必填：名称
- 选填：描述、排序号
- 创建后默认启用

### 6.4 编辑领域

- 可修改：名称、描述、排序号、启用状态

### 6.5 约束

- 禁用领域后，其下版块对普通用户不可见
- 有版块的领域不能直接删除（需先处理版块或软删除）

### 6.6 页面状态

| --- | --- |
| 正常 | 领域列表 |
| 空状态 | "暂无领域" |

## 7. `/admin/users` — 用户管理

### 7.1 功能

admin 管理所有用户角色与状态（仅 admin 可访问）。

### 7.2 用户列表

- 展示所有用户（分页）
- 搜索：按用户名或邮箱搜索
- 筛选：按角色（user / reviewer / admin）、按状态（active / disabled）
- 每项展示：头像、用户名、邮箱、角色、状态、注册时间
- 操作：编辑角色、启用/禁用

### 7.3 编辑用户

- 可修改：角色（user / reviewer / admin）、状态（active / disabled）
- 不可修改：用户名、邮箱
- 操作日志记录（TODO：是否记录操作审计后续确认）

### 7.4 约束

- admin 不能禁用自己
- 修改角色后立即生效，用户下次请求时 JWT 中角色更新

### 7.5 页面状态

| 状态 | 展示 |

## 8. `/admin/tags` — Tag 管理

### 8.1 功能

admin 管理全站 Tag（仅 admin 可访问）。

### 8.2 Tag 列表

- 展示所有 Tag（分页，按使用次数倒序）
- 搜索：按 Tag 名称搜索
- 每项展示：Tag 名称、使用次数
- 操作：编辑名称、合并、删除

### 8.3 合并 Tag

**流程**：
1. admin 选择多个 Tag（如"go"、"golang"、"go语言"）
2. 选择合并目标（保留哪个 Tag 名称）
3. 确认合并 → 所有关联文章更新为使用目标 Tag
4. 被合并的 Tag 可保留或删除

**约束**：
- 合并操作不可撤销（需二次确认）
- 被删除的 Tag 从所有文章的 Tag 列表中移除

### 8.4 编辑/删除 Tag

- 可单独修改 Tag 名称
- 可删除零使用的 Tag
- 删除有使用次数的 Tag 需二次确认（会从所有文章移除）

### 8.5 页面状态

| 状态 | 展示 |
| --- | --- |
| 正常 | Tag 列表（分页+搜索） |
| 空状态 | "暂无 Tag" |
| 搜索无结果 | "未找到匹配 Tag" |
| 合并确认 | 弹窗：展示待合并 Tag + 目标 Tag + 二次确认 |
| 无权访问 | 403（非 admin） |

## 9. `/admin/reports` — 举报管理

### 9.1 功能

审核员处理用户举报。

### 9.2 举报列表

- 展示所有举报记录（分页，时间倒序）
- 筛选：按状态（pending / resolved）、按举报原因
- 每项展示：举报人、被举报文章、举报原因、补充说明、举报时间、处理状态
- 操作：查看被举报文章 → 下架 或 驳回举报

### 9.3 处理举报

**下架文章**：
1. 审核员查看举报详情和被举报文章
2. 确认违规 → 下架文章（`published` → `archived`）
3. 举报标记为"已处理"
4. 可填写处理说明

**驳回举报**：
1. 审核员认为文章无违规
2. 举报标记为"已驳回"
3. 可填写驳回说明

### 9.4 页面状态

| 状态 | 展示 |
| --- | --- |
| 正常 | 举报列表（分页+筛选） |
| 空状态 | "暂无举报" |
| 加载失败 | 错误提示 + 重试 |
| 无权访问 | 403 |
