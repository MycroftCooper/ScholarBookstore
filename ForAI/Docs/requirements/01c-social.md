# 社交模块需求文档

评论、通知、举报与事后审查。

---

## 1. 模块概述

| 子模块 | 说明 |
| --- | --- |
| 评论 | 文章下顶级评论与回复，支持投票与排序 |
| 通知 | 站内通知（评论通知 + 回复通知 + 收藏通知） |
| 举报 | 用户举报违规文章 |
| 事后审查 | 审核员主动巡检下架 |

本模块覆盖 3 个页面 + 评论区内嵌于文章详情页：

| 路由 | 页面 | 需要登录 |
| --- | --- | --- |
| `/me/comments` | 我的评论 | 是 |
| `/me/notifications` | 我的通知 | 是 |
| 评论区（文章详情页内） | 文章详情页 `/articles/[id]` 内嵌 | 查看需登录，操作需登录 |
| 举报入口（文章详情页内） | 文章详情页内嵌"举报"按钮 | 是 |

---

## 2. 评论与回复

### 2.1 核心规则

| 规则 | 说明 |
| --- | --- |
| 可见性 | **游客不可查看评论区**。只有登录用户可查看和参与评论 |
| 评论范围 | 仅 `published` 且未删除文章可评论 |
| 顶级评论 | 直接针对文章发表 |
| 回复评论 | 回复某条顶级评论或其他回复 |
| 结构 | 两层展示：顶级评论 + 该评论下的回复列表 |
| 无需审核 | 评论和回复发布即展示，无需审核 |
| 内容约束 | 纯文本，不解析 Markdown，不嵌入图片 |

### 2.2 加载方式

- 评论区采用**无限滚动**加载
- 初始加载前 N 条顶级评论（含各自前 M 条回复）
- 滚动到底部自动加载更多
- 回复列表过长时，每层仅展示前几条，提供"展开更多"按钮

### 2.3 投票（点赞 / 点踩）

| 操作 | 规则 |
| --- | --- |
| 点赞 | 登录用户可对任意评论（含顶级评论和回复）点赞 |
| 点踩 | 登录用户可对任意评论点踩 |
| 取消 | 再次点击可取消自己的投票 |
| 切换 | 点赞后点踩 → 自动取消赞，改为踩（反之亦然） |
| 计数 | 每条评论展示点赞数和点踩数（净赞 = 赞数 - 踩数） |
| 自己 | 不可对自己的评论投票 |

### 2.4 评论排序

支持两种排序方式，用户可切换：

| 排序 | 规则 |
| --- | --- |
| **最新优先** | 按创建时间倒序，最新评论在前面 |
| **最热优先** | 按热度排序：净赞数高 + 子回复数多的在前 |

- 排序切换仅影响顶级评论的顺序
- 子回复始终按时间正序展示

### 2.5 评论列表展示

**展示方式**（文章详情页内）：

- 排序方式切换按钮（最新 / 最热）
- 顶级评论按所选排序展示
- 每条评论展示：评论者头像（可点击进入作者主页）、用户名（可点击）、发布时间、内容、点赞/点踩按钮和计数、回复按钮
- 被隐藏评论：对本人显示"评论已被隐藏"，对他人不显示
- 已删除评论：显示"评论已删除"

### 2.6 发表顶级评论

**入口**：文章详情页评论输入框（需登录）

**流程**：
1. 用户输入评论内容 → 提交
2. 后端校验文章为 `published` → 创建评论
3. 如果评论他人的文章 → 创建 `article_comment` 通知
4. 如果评论自己的文章 → 不创建通知
5. 新评论立即展示（置顶或按当前排序规则）

**校验**：
- 评论内容必填，最长 2000 字

### 2.7 回复评论

**入口**：每条评论旁的"回复"按钮

**流程**：
1. 点击回复 → 展示回复输入框
2. 用户输入 → 提交
3. 后端校验 → 创建回复（记录 `parent_id` 和 `reply_to_user_id`）
4. 如果回复他人 → 创建 `comment_reply` 通知
5. 如果回复自己 → 不创建通知

**校验**：
- 被回复评论必须存在、未删除、可见
- 被回复评论所属文章必须为 `published`
- 回复内容最长 2000 字

### 2.8 删除与隐藏评论

| 操作者 | 权限 |
| --- | --- |
| 评论作者 | 可软删除自己的评论或回复 |
| reviewer / admin | 可软删除或隐藏任意评论或回复 |

- 被隐藏评论对普通用户不可见（评论者本人显示"评论已被隐藏"）
- 已删除评论展示"评论已删除"
- 有回复的评论被删除后，子回复展示"原评论已删除"

### 2.9 页面状态（评论区）

| 状态 | 展示 |
| --- | --- |
| 正常 | 排序切换 + 评论列表 + 输入框 |
| 未登录 | 不展示评论区，提示"登录后查看评论" |
| 空状态 | "暂无评论，来写第一条评论吧" |
| 评论提交中 | 按钮 loading |
| 评论失败 | 保留输入内容 + 错误提示 |
| 加载更多 | 滚动底部 loading 指示器 |

## 3. 站内通知

### 3.1 通知类型

| 类型 | 触发条件 | recipient |
| --- | --- | --- |
| `article_comment` | 他人对自己的文章发表顶级评论 | 文章作者 |
| `comment_reply` | 他人回复自己的评论 | 被回复者 |
| `article_bookmark` | 他人收藏自己的文章 | 文章作者 |

### 3.2 通知创建规则

- 必须由后端在创建评论/回复/收藏的**同一事务中**完成
- 前端不能直接创建通知
- 操作自己的内容时不创建通知
- 操作已禁用用户的内容时，不创建通知

### 3.3 `/me/notifications` — 我的通知

**展示内容**：
- 通知列表（分页，时间倒序）
- 每项展示：触发人头像（可点击）、用户名（可点击）、通知类型文案、关联文章标题、时间、已读/未读状态
- 筛选：全部 / 未读

**通知文案**：
- `article_comment`："xxx 评论了你的文章《文章标题》"
- `comment_reply`："xxx 回复了你的评论"
- `article_bookmark`："xxx 收藏了你的文章《文章标题》"

**用户操作**：
- 点击通知 → 标记已读 + 跳转对应文章
- "全部标记已读"
- 单条"标记已读"

**页面状态**：

| 状态 | 展示 |
| --- | --- |
| 正常 | 通知列表 |
| 未登录 | 跳转登录页 |
| 空状态 | "暂无通知" |
| 加载失败 | 错误提示 + 重试 |

### 3.4 未读通知数量

- 顶部导航栏展示未读通知数（数字徽章）
- 仅已登录时显示

## 4. 举报

### 4.1 流程

1. 登录用户在文章详情页点击"举报"
2. 弹出举报表单：选择举报原因（必选）、补充说明（可选）
3. 提交 → 后端创建举报记录
4. 审核员在后台处理举报（详见 `01e-admin.md`）

### 4.2 举报原因

| 选项 | 说明 |
| --- | --- |
| 违规内容 | 违反社区规范 |
| 垃圾广告 | 推广、广告内容 |
| 侵权内容 | 侵犯他人版权 |
| 其他 | 需填写补充说明（必填） |

### 4.3 约束

- 只能举报 `published` 文章
- 同一用户对同一文章只能有一条**未处理**的举报
- 举报不影响文章正常展示（除非被下架）

### 4.4 页面状态（举报弹窗）

| 状态 | 展示 |
| --- | --- |
| 正常 | 举报原因选择 + 补充说明 + 提交 |
| 未登录 | 提示"请先登录" |
| 已举报 | 提示"你已举报过该文章，等待处理" |
| 提交中 | 按钮 loading |
| 提交成功 | 提示"举报已提交" |

## 5. 事后审查

审核员可在文章详情页或管理后台主动下架已发布文章（详见 `01e-admin.md`）。

## 6. `/me/comments` — 我的评论

### 6.1 展示

- 用户发表过的评论和回复列表（分页，时间倒序）
- 每项展示：评论内容摘要、所属文章标题、发布时间、点赞数
- 操作：删除自己的评论

### 6.2 页面状态

| 状态 | 展示 |
| --- | --- |
| 正常 | 评论列表 |
| 未登录 | 跳转登录页 |
| 空状态 | "暂无评论" |
| 加载失败 | 错误提示 + 重试 |

---

## 7. 权限约束

- 游客不可查看评论区（整体不可见）
- 创建评论/回复/收藏和通知必须同一事务
- 前端不可直接创建通知
- 评论和回复默认不审核
- 不可对自己的评论投票
- 举报仅限已发布文章，不可重复举报

---

## 8. 验收标准

### 评论
- [ ] 游客在文章详情页不可见评论区
- [ ] 登录用户可查看评论区
- [ ] 评论区无限滚动加载
- [ ] 评论支持切换"最新"/"最热"排序
- [ ] 评论支持点赞和点踩，可取消和切换
- [ ] 不可对自己的评论投票
- [ ] 评论他人文章时作者收到 `article_comment` 通知
- [ ] 回复他人评论时被回复者收到 `comment_reply` 通知
- [ ] 评论自己/回复自己不创建通知
- [ ] 用户可删除自己的评论/回复
- [ ] 空评论区展示提示

### 通知
- [ ] 通知与评论/回复/收藏在同一事务中创建
- [ ] 用户只能查看自己的通知
- [ ] 用户可标记单条/全部已读
- [ ] 未读数量在导航栏正确展示
- [ ] 他人收藏文章时作者收到 `article_bookmark` 通知
- [ ] 收藏自己的文章不创建通知

### 举报
- [ ] 登录用户可在文章详情页举报
- [ ] 必须选择举报原因
- [ ] 不可重复举报（未处理状态下）

### 事后审查
- [ ] 审核员可在文章详情页直接下架文章
