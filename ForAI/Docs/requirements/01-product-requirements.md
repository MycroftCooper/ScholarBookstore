# 产品需求文档

> 本文档是产品范围的**唯一权威来源**。API 设计、数据库、页面路由、权限安全等技术细节在各专题文档中维护，本文档只描述"做什么"。

## 1. 产品定位

ScholarBookstore 是一个面向开发者的**知识库社区网站**。项目代号中的"Bookstore"与实体书无关，不涉及图书交易。

产品结构为三级：

```text
网站 → 领域 → 版块 → 文章
```

- **领域**：一级分类（如"后端开发"、"游戏开发"、"前端开发"）
- **版块**：所属领域下的二级分类（如领域"游戏开发"下的"游戏客户端"、"游戏服务器"、"游戏策划"）
- **文章**：属于某个版块的技术文章

不做的功能：

- 论坛、帖子、贴吧式主题帖系统（讨论需求收敛到文章评论区）
- 实时聊天、私信
- 付费会员、支付、订单
- 积分、等级、徽章、任务系统
- 复杂推荐算法
- 原生 iOS / Android App

## 2. 目标用户

| 角色 | 说明 |
| --- | --- |
| guest | 未登录用户。浏览公开内容，可搜索文章 |
| user | 已登录普通用户。投稿、评论、回复、管理个人资料、接收通知 |
| reviewer | 审核员。审核文章、管理内容、处理举报、查看数据看板 |
| admin | 管理员。拥有全部权限，包括领域/版块/用户管理 |

游客不入库，由未登录状态表示。

## 5. 核心流程摘要

### 5.1 注册登录

用户注册（用户名/邮箱/密码）→ 登录获取 JWT（httpOnly Cookie）。忘记密码时联系管理员处理。详见 `01a-auth.md`。

### 5.2 草稿 → 投稿 → 审核 → 发布

用户写文章 → 可保存草稿（`draft`） → 提交审核（`pending_review`）→ 审核员通过（`published`）或拒绝（`rejected`）→ 拒绝可修改重提。详见 `01b-content.md`。

### 5.3 已发布文章修订重审

作者发起修订 → 创建修订版（原文保持 `published`）→ 修订版进 `pending_review` → 审核通过后替换原文。详见 `01b-content.md`。

### 5.4 收藏

登录用户收藏文章（可选收藏夹分类）→ 文章详情展示收藏数 → 被收藏时文章作者收到通知。详见 `01b-content.md`。

### 5.5 评论与回复

登录用户在 `published` 文章下评论/回复 → 支持点赞点踩 → 支持按时间/热度排序 → 游客不可见评论区。详见 `01c-social.md`。

### 5.6 站内通知

文章被评论/被收藏 → 通知文章作者。评论被回复 → 通知被回复者。用户可标记已读。详见 `01c-social.md`。

### 5.7 搜索

全文搜索已发布文章（标题+摘要+正文），支持按 Tag 筛选。详见 `01d-discover.md`。

### 5.8 举报与事后审查

用户举报已发布文章 → 审核员在后台处理举报 → 审核员也可主动巡检下架违规内容。详见 `01c-social.md`。

### 5.9 关注与取关

登录用户在作者主页或文章页点击"关注"→ 关注该作者 → 该作者发布新文章时，所有关注者收到 `followee_article` 通知。再次点击"已关注"取关。用户可在个人中心查看"我关注的"和"关注我的"列表。详见 `01c-social.md`。

### 5.10 图片上传

登录用户上传图片（仅限 jpeg/png/webp/gif，≤5MB）→ 后端校验 → 存本地 uploads → 返回 URL 供 Markdown 引用。
