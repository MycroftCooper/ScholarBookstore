# 内容模块需求文档

领域、版块、文章——三级结构的完整需求。含作者主页、收藏、TOC、Tag。

---

## 1. 模块概述

| 模块 | 说明 |
| --- | --- |
| 领域 | 一级分类，如"后端开发"、"游戏开发" |
| 版块 | 二级分类，属于某个领域，如领域"游戏开发"下的"游戏客户端" |
| 文章 | 属于某个版块的技术文章，支持 Markdown |

本模块覆盖 11 个页面：

| 路由 | 页面 | 需要登录 |
| --- | --- | --- |
| `/domain` | 领域列表 | 否 |
| `/domain/[id]` | 领域详情（该领域下版块列表） | 否 |
| `/modules/[slug]` | 版块详情（文章列表） | 否 |
| `/articles/[id]` | 文章详情（含 TOC + 收藏入口） | 阅读否 |
| `/authors/[username]` | 作者公开主页 | 否 |
| `/me/submit` | 投稿（含草稿保存） | 是 |
| `/me/articles` | 我的投稿 | 是 |
| `/me/drafts` | 我的草稿 | 是 |
| `/me/articles/[id]/edit` | 编辑投稿 | 是 |
| `/me/bookmarks` | 我的收藏（含收藏夹管理） | 是 |
| `/discover` | 搜索（详见 `01d-discover.md`） | 否 |

## 2. 文章状态流转

### 2.1 状态枚举

| 状态 | 说明 |
| --- | --- |
| `draft` | 草稿，仅作者可见 |
| `pending_review` | 待审核 |
| `published` | 已发布，公开可见 |
| `rejected` | 审核拒绝 |
| `archived` | 已下架 |

### 2.2 状态流转图

```text
draft ──────→ pending_review ──────→ published ──────→ archived
  ↑                 │                      ↑               │
  │                 │                      │               │
  └─── 继续编辑 ←───┘                      │               │
  │                                        │               │
  └── rejected ──────→ pending_review     │               │
                                             │               │
                                    修订审核通过 ─────────────┘
                                    (修订版替换原文)
```

### 2.3 流转规则

| 操作 | 触发者 | 状态变化 | 备注 |
| --- | --- | --- | --- |
| 保存草稿 | 作者 | → `draft` | 新建或继续编辑 |
| 提交审核 | 作者 | `draft` → `pending_review` | |
| 审核通过 | reviewer/admin | `pending_review` → `published` | 设置 `published_at`；同时为所有关注作者的活跃用户创建 `followee_article` 通知 |
| 审核拒绝 | reviewer/admin | `pending_review` → `rejected` | 必须填写 `review_note` |
| 修改重提 | 作者 | `rejected` → `pending_review` | 清空旧审核说明 |
| 继续编辑草稿 | 作者 | 保持 `draft` | |
| 下架 | reviewer/admin | `published` → `archived` | 文章不再公开可见 |
| 恢复 | reviewer/admin | `archived` → `published` | |
| 发起修订 | 作者 | 创建修订版（原文保持 `published`） | 见第 6 节 |

## 4. 版块页面

## 5. 文章详情页 `/articles/[id]`

### 5.1 展示内容

- 文章标题
- 作者头像+用户名（可点击进入作者主页）
- 所属版块、发布时间、字数、浏览量、收藏数、预计阅读时长、修订版本数
- Tag 标签列表（可点击跳转搜索，每个 Tag 显示使用次数）
- **TOC 目录**：基于 Markdown 正文 h2/h3 标题自动生成，侧边栏或文章顶部展示，点击锚点跳转
- Markdown 正文（含图片）
- 登录后可见评论区（详见 `01c-social.md`）

### 5.2 用户操作

- 阅读文章
- 登录后评论和回复（含点赞点踩，详见 `01c-social.md`）
- 登录后收藏文章（选择收藏夹）
- **作者**：可见"编辑"按钮（发起修订）
- **所有登录用户**：可见"关注/取关"按钮（关注文章作者）
- **所有登录用户**：可见"举报"按钮（举报他人文章）

### 5.3 页面状态

| 状态 | 展示 |
| --- | --- |
| 正常 | 文章正文 + TOC + 评论区（登录后可见） |
| 文章不存在/未发布/已删除 | 404 |
| 加载中 | 骨架屏 |
| 加载失败 | 错误提示 + 重试 |

### 5.4 移动端

- Markdown 内容不横向溢出
- 图片自适应容器宽度
- 代码块可横向滚动
- TOC 可折叠

## 6. 已发布文章修订重审

### 6.1 流程

1. 文章作者在文章详情页点击"编辑"
2. 跳转 `/me/articles/[id]/edit`，展示当前文章内容
3. 作者修改标题、摘要、正文、Tag
4. 提交后，**创建修订版**（新记录，状态 `pending_review`）
5. **原文保持 `published` 状态，继续公开可见**
6. 修订版进入审核流程
7. 审核通过 → 修订版内容替换原文，修订版本数 +1
8. 审核拒绝 → 修订版变 `rejected`，原文不受影响

### 6.2 约束

- 同一篇文章同时只能有一个待审核修订版（`pending_review` 或 `rejected`）
- 已有待审核修订版时，再次点击"编辑"进入该修订编辑页
- 修订版通过后，旧修订记录保留

## 8. 文章收藏

### 8.1 收藏操作

- 登录用户在文章详情页点击"收藏"按钮
- 弹出收藏夹选择器：选择已有收藏夹 或 新建收藏夹
- 收藏夹名称最长 30 字
- 同一文章可加入多个收藏夹
- 取消收藏：进入收藏夹，移除文章

### 8.2 收藏数展示

- 文章详情页展示"被收藏 X 次"
- 文章卡片展示收藏数

### 8.3 收藏通知

- 他人收藏你的文章 → 创建 `article_bookmark` 通知
- 收藏自己的文章 → 不创建通知

### 8.4 `/me/bookmarks` — 我的收藏

**展示内容**：
- 收藏夹列表（左侧或顶部）
- 选中收藏夹后展示其中文章列表（右侧或下方）
- 支持创建、重命名、删除收藏夹
- 默认收藏夹"默认收藏夹"，不可删除

**操作**：
- 从收藏夹中移除文章
- 批量移动文章到其他收藏夹

**页面状态**：

| 状态 | 展示 |
| --- | --- |
| 正常 | 收藏夹 + 文章列表 |
| 未登录 | 跳转登录页 |
| 空收藏夹 | "收藏夹为空" |
| 无收藏夹 | 展示默认收藏夹（空） |

## 9. 投稿页 `/me/submit`

### 9.1 功能

- 版块选择（下拉选择启用版块）
- 标题输入
- 摘要输入（可选，最长 300 字）
- Markdown 正文编辑
- 图片上传（插入 Markdown）
- Tag 添加（见第 14 节）
- 草稿保存
- 提交审核

### 9.2 入口

- 用户可从版块详情页、导航栏等位置的投稿按钮跳转至此
- 不在 URL 中预设版块参数，用户在表单内自行选择

### 9.3 草稿保存

- 用户点击"保存草稿" → 文章进入 `draft` 状态
- 草稿可在 `/me/drafts` 中找到并继续编辑
- 草稿可多次保存

### 9.4 提交审核

- 用户点击"提交审核" → 文章进入 `pending_review` 状态
- 必填：版块、标题、正文
- 选填：摘要、Tag

### 9.5 页面状态

| 状态 | 展示 |
| --- | --- |
| 正常 | 完整投稿表单 |
| 未登录 | 跳转登录页 |
| 无可投稿版块 | 提示"暂无可投稿版块" |
| 图片上传中 | 上传进度或 loading |
| 图片上传失败 | 文件校验错误提示 |
| 提交中 | 按钮 loading |
| 提交成功 | 跳转 `/me/articles` 或提示 |
| 校验失败 | 对应字段红色提示 |
| 自动草稿 | 内容未保存时离开页面提示 |

## 12. 编辑投稿 `/me/articles/[id]/edit`

### 12.1 功能

复用页面，覆盖三种场景：

| 场景 | 文章状态 | 操作 |
| --- | --- | --- |
| 编辑草稿 | `draft` | 修改后保存草稿或提交审核 |
| 修改拒绝稿 | `rejected` | 查看拒绝原因 → 修改 → 提交（重新进 `pending_review`） |
| 修订已发布文章 | `published` | 查看当前内容 → 修改 → 提交（创建修订版进 `pending_review`） |

### 12.2 约束

- 仅作者可编辑自己的文章
- 版块**只读展示**，不可修改
- `archived` 文章不支持编辑
- 修改后保存触发对应的状态流转

### 12.3 页面状态

| 状态 | 展示 |
| --- | --- |
| 正常 | 编辑表单，预填现有内容 |
| 未登录 | 跳转登录页 |
| 无权访问 | 403 |
| 文章不存在 | 404 |
| 已归档文章 | 提示"已归档文章不可编辑" |
| 图片上传中 | 上传进度或 loading |
| 提交中 | 按钮 loading |
| 提交成功 | 跳转 `/me/articles` |

## 14. Tag 标签系统

### 14.1 规则

- 用户投稿时可自由输入 Tag
- 每篇文章最多 9 个 Tag
- Tag 不预设分类，无层级关系

### 14.2 Tag 使用次数

- 用户输入或选择 Tag 时，Tag 旁显示该 Tag 当前被使用次数
- 引导用户选择相近概念中使用次数最高的 Tag
- 示例：输入"go"时，推荐列表显示 `Go (42次)` / `golang (18次)` / `go语言 (3次)`

### 14.3 展示位置

1. **文章详情页顶部**：标题下方展示 Tag，每个 Tag 显示使用次数
2. **文章卡片**：列表中的文章卡片展示 Tag
3. **发现页聚合**：按 Tag 聚合展示热门 Tag 下的文章
4. **搜索结果**：支持按 Tag 筛选

### 14.4 Tag 合并（管理端）

- admin 可在管理后台将多个 Tag 合并为一个（如 "go"/"golang"/"go语言" → "Go"）
- 合并后所有关联文章的 Tag 更新为合并目标
- 被合并的 Tag 可以保留或删除
- 详见 `01e-admin.md`

## 15. 文章元数据

### 15.1 字段说明

| 字段 | 计算方式 | 展示位置 |
| --- | --- | --- |
| 字数 | Markdown 正文字符数 | 文章详情、文章卡片、我的投稿列表 |
| 浏览量 | 每次页面访问 +1（简单计数） | 文章详情、文章卡片、我的投稿列表 |
| 预计阅读时长 | 字数 / 300 字/分钟（向上取整） | 文章详情、文章卡片 |
| 修订版本数 | 已通过修订次数 + 1（初版） | 文章详情 |
| 收藏数 | 收藏该文章的用户数 | 文章详情、文章卡片、我的投稿列表 |

### 15.2 浏览量规则

- 每次 `/articles/[id]` 页面加载 +1
- 第一版不去重（同一用户多次访问也计数）
- 用于发现页热点聚合排序（浏览量 + 时间衰减加权）

---

## 16. 图片上传

### 16.1 规则

- 仅限登录用户上传
- 允许 MIME：`image/jpeg`、`image/png`、`image/webp`、`image/gif`
- 最大 5MB
- 后端生成存储文件名
- 文件存本地 `uploads` 目录
- 返回公开访问 URL，前端插入 Markdown

---

## 17. 权限约束

- 游客只能查看 `published` 且未删除的文章
- 游客不可查看评论
- 作者只能编辑自己的文章
- 收藏仅限登录用户
- 审核权限仅限 reviewer/admin（详见 `01e-admin.md`）
- 领域和版块管理仅限 admin（详见 `01e-admin.md`）
- 后端必须校验所有权限，前端仅做体验控制
