# 产品需求文档

## 1. 产品定位

本项目是一个面向个人长期维护的知识库社区网站

第一版只聚焦“版块下沉淀技术文章”的核心闭环：注册登录、按版块投稿、审核发布、阅读文章、评论文章、回复评论、通知文章作者和被回复用户、个人中心查看自己的内容和通知。

第一版不做论坛、帖子、贴吧式主题帖系统。讨论需求先收敛到文章评论区。

## 2. 目标用户

- 游客：浏览公开版块、已发布文章、禁止浏览文章评论和评论回复，也禁止评论。
- 普通用户：注册登录后投稿文章、发表评论、回复评论、接收通知、管理自己的投稿和评论。
- 审核员：审核用户投稿，处理明显违规的文章、评论和评论回复。
- 管理员：管理版块、用户角色、审核员权限、全站内容和系统配置。

## 3. MVP 范围

MVP 必须包含：

- 用户注册、登录、退出登录。
- 基于 JWT + httpOnly Cookie 的登录态。
- 版块列表与版块详情。
- 用户按版块投稿 Markdown 技术文章。
- 投稿文章进入审核流程，审核通过后才公开。
- 已发布文章详情页。
- 每篇文章的评论区。
- 评论支持回复评论。
- 被回复用户收到站内通知。
- 个人中心：我的投稿、我的评论、我的通知、基础账户信息。
- 图片上传，第一版使用本地 `uploads`。
- 管理后台基础能力：文章审核、版块管理、内容隐藏或软删除。
- Web 支持。PWA 安装与离线能力不纳入当前 MVP，后续单独迭代。

## 4. 用户角色

| 角色 | 说明 |
| --- | --- |
| guest | 未登录用户，只能浏览公开内容 |
| user | 已登录普通用户，可投稿、评论、回复评论、查看个人中心 |
| reviewer | 审核员，可审核文章与处理内容 |
| admin | 管理员，拥有系统管理权限 |

角色枚举必须与 `06-permission-security.md` 保持一致。

## 5. 核心流程

### 5.1 注册登录流程

1. 用户提交用户名、邮箱、密码。
2. 后端校验唯一性和密码强度。
3. 使用 bcrypt 保存密码哈希。
4. 登录成功后后端签发 JWT，并通过 httpOnly Cookie 返回。
5. 前端不允许将 JWT 保存到 localStorage 或 sessionStorage。

### 5.2 投稿审核流程

1. 登录用户选择版块。
2. 用户填写标题、摘要、Markdown 正文并上传图片。
3. 提交后文章状态为 `pending_review`。
4. 审核员或管理员查看待审核列表。
5. 审核通过后状态变为 `published`，文章公开可见。
6. 审核拒绝后状态变为 `rejected`，作者可在个人中心查看拒绝原因。
7. 作者修改 `rejected` 文章后，文章重新进入 `pending_review`。

### 5.3 评论与回复流程

1. 游客可查看已发布文章下的可见评论和回复。
2. 登录用户可在已发布文章下发表评论。
3. 登录用户可回复某条评论。
4. 顶级评论他人文章时，后端创建通知给文章作者。
5. 回复评论时，后端创建通知给被回复评论的作者。
6. 如果评论或回复自己，不创建通知。
7. 评论作者可软删除自己的评论或回复。
8. 审核员和管理员可隐藏或软删除违规评论和回复。

### 5.4 通知流程

1. 用户文章收到他人顶级评论时，系统创建一条站内通知。
2. 用户评论收到他人回复时，系统创建一条站内通知。
3. 用户在个人中心查看通知列表。
4. 用户可将通知标记为已读。
5. MVP 只做站内通知，不做邮件、短信、浏览器推送。

### 5.5 图片上传流程

1. 登录用户上传图片。
2. 后端校验文件类型、大小和归属场景。
3. 文件保存到本地 `uploads`。
4. 数据库记录图片 URL、上传者、关联文章。
5. Markdown 正文引用后端返回的图片 URL。

## 6. 明确不做的功能

第一版明确不做：

- 原生 iOS / Android App。
- 论坛、帖子、贴吧式主题帖、帖子回复。
- 实时聊天、私信系统。
- 邮件通知、短信通知、浏览器推送通知。
- 复杂推荐算法。
- 积分、等级、徽章、任务系统。
- 付费会员、支付、订单。
- 多租户、组织空间。
- 复杂全文搜索引擎，如 Elasticsearch、Meilisearch。TODO：是否使用 PostgreSQL full text search 后续确认。
- PWA 安装、离线缓存、浏览器推送。
- Redis 缓存。
- GraphQL。
- 微服务架构。
- 第三方 BaaS 替代后端，如 Supabase、Firebase。
- 云对象存储强依赖。第一版仅使用本地 `uploads`，后续可迁移到 Cloudflare R2 或 S3。

## 7. 成功标准

- 普通用户能完成注册、登录、投稿、查看审核状态。
- 审核员能审核文章并给出拒绝原因。
- 游客能浏览公开版块、已发布文章、评论和评论回复。
- 登录用户能评论文章、回复评论。
- 被回复用户能在个人中心看到通知。
- 管理员能管理版块和处理违规内容。
- 单人维护时部署、备份、排查问题的复杂度可控。
