# TODO 功能清单

本文件只记录“当前还没实现、但后续可能要做”的功能缺口。

维护规则：

- 功能实际完成并验证后，必须从本文档移除对应条目。
- 已明确不做的能力不要放入本文档。
- 新增 TODO 时写清楚所属模块、当前状态、需要补齐的后端/前端/文档工作。
- 不要把临时 bug、个人备忘或一次性排查事项放进本文档。

## 搜索与发现

- [ ] 搜索结果高亮摘要
  - 当前状态：公开文章搜索已升级为 PostgreSQL full text search，并有 GIN 索引和相关查询排序；搜索结果仍只展示普通标题/摘要，不返回命中片段。
  - 需要补齐：后端 `ts_headline` 或等价命中片段、前端安全高亮规则、无摘要文章的降级展示和测试。

## 管理与安全

- [ ] 登录页扩展登录能力
  - 当前状态：前端登录页展示“记住我、忘记密码”入口，`/forgot-password` 已有前端页面壳；实际登录仍只接现有 `/auth/login` 邮箱+密码接口，找回密码不会发送邮件。第三方登录已明确不做。
  - 需要补齐：确认用户名登录规则、记住登录态/续期策略、找回密码令牌/邮件发送/重设密码 API、前端接入与错误提示。

- [ ] 用户角色管理 UI
  - 当前状态：已有 `/admin/users` 和 `/admin/roles`，可管理全局 admin/reviewer、领域领主和版块版主，并接入审计日志；权限矩阵仍偏静态说明，角色申请类待办尚未形成完整流程。
  - 需要补齐：+角色申请/审批流程、权限矩阵与后端权限规则的同步校验、变更备注持久化、危险操作二次确认和更细测试。

- [ ] 操作审计日志覆盖增强
  - 当前状态：已存在 `audit_logs` 表、后台 `/admin/audit-logs` 页面和多处后台操作写入；待办任务处理、部分举报处理、自动任务和失败操作的审计覆盖仍需逐项核对。
  - 需要补齐：审计动作命名统一、所有高风险入口覆盖清单、失败/拒绝类事件是否记录的策略、保留/导出策略和审计写入测试。

- [ ] 内容敏感词或审核队列
  - 当前状态：文章投稿进入审核待办；评论举报类型已在任务模型和前端筛选中预留，但评论/回复发布本身默认不走敏感词或审核队列。
  - 需要补齐：是否引入敏感词、评论审核队列、评论举报创建入口、命中后的用户提示与管理入口。

- [ ] Refresh token 与 token 轮换
  - 当前状态：使用 JWT + httpOnly Cookie，未实现 refresh token 轮换机制。
  - 需要补齐：登录态续期策略、失效策略、服务端存储或无状态方案评估。

## 个人资料与设置

- [ ] 扩展个人资料字段
  - 当前状态：`/me/profile` 只持久化头像、Bio、学校和公司；用户名、邮箱、地区、技术标签和展示设置在设置页中仅作只读、静态展示或空状态。
  - 需要补齐：后端用户资料字段或独立 profile 表、PATCH API、公开作者主页展示规则、前端表单保存逻辑和迁移文档。

- [ ] 用户简历
  - 当前状态：作者公开主页已预留“简历”tab，但没有数据模型、编辑入口和公开展示内容。
  - 需要补齐：简历字段/表结构、编辑页入口、公开作者主页展示规则、权限与隐私边界。

- [x] 举报处理第一期闭环
  - 当前状态：文章、评论、用户主页举报均可提交；后台 `/admin/tasks` 已统一展示 `content_report`、`comment_report`、`user_report`；处理动作支持忽略、单内容下架/隐藏、禁用账号、限制关注、禁止发文、禁止评论，并写入审计日志。
  - 后续补齐：批量下架/隐藏、处罚撤销、处罚历史页、申诉、恶意举报识别和自动风控。详见 `14-moderation-reports.md`。

## 消息与通知
